PRMTR Visual Language

The complete visual identity for the PRMTR commercial-intelligence platform — foundations, every component and state, overlays, and section mockups. This document is the living render of design.md. Everything you build must follow it; anything missing must be composed from these same foundations.

01

Principles

One screen, one decision
Lead with the answer — the KPI, the ranked action — then progressive detail.
Numbers are the hero
Big, tabular, monospaced figures. Labels quiet; values loud.
Color = consequence
Green money/safe, red loss/blocked, amber caution, lime "act now". Never decorative.
Square & serious
Radius is always 0. Calm off-white canvas, hairline borders, soft shadows.

Governance

The rule for missing components

When development needs a component this system hasn't specified yet, do not invent a new style. Compose it from these foundations — radius 0, the 40px control height, 1px --color-border, the --space-* scale, the --shadow-* elevation, the type scale, and the semantic color roles — using the nearest documented component as precedent. Then add it back to design.md and this page so the next person inherits it. Additive only — never fork the palette or the type scale.

Always
  • Consume semantic tokens
  • Square corners (radius 0)
  • Mono + tabular for figures
  • Color only as consequence
  • Surface the margin floor & 25% cap
Never
  • Hard-code hex or pixels
  • Add rounding or new fonts
  • Decorate with status colors
  • Stack multiple signal accents
  • Show a Run CTA on a breaching action
Naming
3-tier: primitive → semantic → component. kebab-case, role-before-variant-before-state. New tokens override semantics only.
